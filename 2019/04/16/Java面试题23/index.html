  11. 哪些不属于高并发原则()

    A. 无状态设计: 因为有状态可能涉及锁曹, 锁又可能导致并发的串行化;

    B. 保持合理的颗粒, 无论拆分还是服务化, 其实就是服务粒度控制, 控制粒度为了分散请求, 提高并发, 成为了从管理等角度提高可操性

    C. 系统的任何发布必须具有回滚能力

    D. 缓存, 队列, 并发等技巧在高并发设计上可供参考, 但需依场景使用

    B. 成熟的老框架

    C. 开源社区是否活跃

    D. 文档是否丰富

    B. 同步接口提高异步批量接口

    D. 微服务
